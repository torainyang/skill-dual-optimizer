# Skill Quality Rubric

用于评估 skill 设计质量。基于两维度框架，每维度 3 档。

---

## 评分维度

### D1: 上下文与执行可靠性

LLM 能不能稳定执行这个 skill？

- **优秀**：关键约束在首尾高注意力区，无中间遗忘，高噪音时能提取有效信号，多步流程稳定无跳步且依赖显式声明，字数高效（<5000字或合理使用 references），引用文件健康
- **合格**：结构基本合理，但部分重要规则位置偏低或距执行点过远，偶有跳步或遗忘，字数偏高但可接受
- **需修复**：关键规则被埋在中间，高噪音时崩溃，频繁跳步或中间产物丢失或依赖隐式，字数严重超标（>10000字/约500行），引用文件大段重复

### D2: 认知与决策质量

Skill 的输出真正有用吗？

- **优秀**：推理无衰减，无幻影指令，LLM/脚本职责清晰，确认门槛硬编码，Gotchas 覆盖高频错误且有正向示例（≥3:1），输出可直接使用
- **合格**：推理基本可靠但靠后步骤质量下降，偶有幻影指令或 LLM/脚本边界模糊，Gotchas 覆盖主要风险但少量遗漏，输出偶需二次加工
- **需修复**：推理衰减严重，存在诱导编造的指令或 LLM 做确定性操作，确认门槛为软约束，Gotchas 缺失，输出不稳定需大幅修改

---

## 综合判定

- **A（全优秀或无"需修复"且"合格"≤1项）**：可直接使用
- **B（无"需修复"，"合格"≤2项）**：可用，需针对性优化
- **C（"需修复"1项）**：需修复对应维度
- **D（"需修复"≥2项）**：需重构

## 快速失败标志 (Quick-Fail Flags)

以下任一项为真时，标记为需要立即修复：

1. description 无法触发目标场景
2. 工作流中存在必须确认却没有卡住的步骤
3. 存在互相矛盾的指令
4. 存在 LLM 无法执行的幻影指令
5. SKILL.md 超过 5000 字（约 300 行）且未使用 references
6. 引用文件存在大段重复内容
7. Gotchas 完全缺失

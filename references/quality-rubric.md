# Skill Quality Rubric

用于量化评估 skill 设计质量。每个维度 0-5 分，按权重计算综合分。

---

## 评分维度

### 1. 触发准确性 (Trigger Accuracy) — 权重 25%

能否在正确场景被触发，且不误触发。

- 5: description 精准覆盖所有目标场景，包含用户真实会说的关键词，明确排除不适用场景
- 4: description 覆盖主要场景，偶有遗漏或边界模糊
- 3: description 能触发但不够精准，可能与相邻 skill 冲突
- 2: description 过于笼统，频繁误触发或漏触发
- 1: description 基本无法引导正确触发
- 0: 无 description 或完全无关

### 2. 工作流可执行性 (Workflow Executability) — 权重 25%

LLM 能否稳定、正确地按流程执行。

- 5: 步骤清晰有序，确认门槛硬编码，失败有降级策略，输出格式固定
- 4: 步骤清晰，有确认门槛，但缺少降级策略或输出格式偶有不一致
- 3: 步骤基本可执行，但存在跳步风险或确认门槛不够硬
- 2: 步骤模糊，LLM 需要大量自行判断，执行结果不稳定
- 1: 流程混乱，LLM 无法稳定执行
- 0: 无工作流或工作流不可执行

### 3. 结构效率 (Structure Efficiency) — 权重 20%

Token 使用是否高效，渐进式披露是否合理。

- 5: SKILL.md 精简只含核心流程，细节合理拆到 references/scripts/assets，按需加载
- 4: 结构基本合理，少量内容可以进一步拆分
- 3: SKILL.md 偏长但尚可接受，存在一些重复或可拆分内容
- 2: SKILL.md 臃肿，大量内容应拆到 references 但没有
- 1: 所有内容堆在单文件，严重浪费上下文窗口
- 0: 结构混乱，无法有效利用

### 4. 防护完备性 (Guardrail Completeness) — 权重 15%

是否有效防止 LLM 常见错误行为。

- 5: Gotchas 精准覆盖高频错误，有 checklist，正向示例充分，无幻影指令
- 4: 有 Gotchas 且基本覆盖主要风险，少量遗漏
- 3: 有部分防护但不系统，存在明显盲区
- 2: 防护薄弱，LLM 容易犯已知错误
- 1: 几乎无防护
- 0: 无任何防护机制

### 5. 输出质量 (Output Quality) — 权重 15%

输出是否对终端用户真正有用。

- 5: 输出结构固定、可直接使用，有长度控制，支持多样性，方便下游 agent 消费
- 4: 输出结构基本固定，偶有格式不一致
- 3: 输出可用但结构不够稳定，需要用户手动整理
- 2: 输出质量不稳定，经常需要大幅修改
- 1: 输出基本不可用
- 0: 无有效输出

---

## 综合评分计算

```
综合分 = 触发准确性 × 0.25 + 工作流可执行性 × 0.25 + 结构效率 × 0.20 + 防护完备性 × 0.15 + 输出质量 × 0.15
```

## 质量等级

- A (4.0-5.0): 生产就绪，可公开分发
- B (3.0-3.9): 基本可用，需要针对性优化
- C (2.0-2.9): 存在明显问题，需要重构关键部分
- D (<2.0): 需要重新设计

## 快速失败标志 (Quick-Fail Flags)

以下任一项为真时，无论综合分多高，都应标记为需要立即修复：

- description 无法触发目标场景
- 工作流中存在必须确认却没有卡住的步骤
- 存在互相矛盾的指令
- 存在 LLM 无法执行的幻影指令
- SKILL.md 超过 300 行且未使用 references

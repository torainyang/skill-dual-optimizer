## 渐进式披露三种模式

**模式1：高级指南 + 参考** — SKILL.md 概述 + 按需加载 references/。适用于单一领域。

**模式2：按领域分文件** — 多领域 skill 按领域拆 references/，用户问某领域时只读对应文件。避免加载无关上下文。

**模式3：条件详情** — 基础内容在 SKILL.md，高级功能链接到独立文件，只在用户需要时读取。

通用约束：引用距离 SKILL.md 一级深（不要嵌套引用）；超过 100 行的参考文件顶部加目录。

## 反馈循环模式

运行验证器 → 修复错误 → 重复。适用于质量关键的输出。验证器可以是脚本（确定性检查）或参考文档（风格合规比对）。

## 可验证中间输出模式

复杂批量操作时：分析 → **创建计划文件**（如 changes.json）→ **用脚本验证计划** → 执行 → 验证。

适用：批量操作、破坏性更改、复杂验证规则、高风险操作。

## 脚本设计原则

- 脚本应解决错误而非推卸给 LLM（"solve, don't delegate"）
- 配置参数必须自文档化，禁止魔法数字
- SKILL.md 中明确标注脚本是"执行"还是"作为参考读取"
- 确定性操作优先用脚本（文本切片、字数统计、文件合并等），不让 LLM 做

## Skill 模式分类

识别目标 skill 的主模式，主模式决定审查重心。

| 模式 | 核心特征 | 标志 | 审查重心 |
|------|---------|------|---------|
| Tool Wrapper | 封装外部工具/API，核心是路由 | 有 API 文档、curl 示例 | description 触发准确性、API 参数完整性、Gotchas 覆盖不在收录范围的场景 |
| Generator | 根据输入生成结构化输出 | 有输出模板、格式要求 | 输出格式稳定性、确认门槛（未确认输入不生成）、幻影指令 |
| Reviewer | 审查输入并输出诊断 | 有评分标准、问题分类 | 评分标准可执行性、D1/D2 框架自洽性、负向引导风险 |
| Inversion | 反向操作（从输出反推输入） | 有逆向映射规则 | 逆向映射完备性、边界条件覆盖、确认门槛 |
| Pipeline | 多步骤顺序执行，步骤间有依赖 | 有阶段门槛、中间产物 | 步骤依赖显式声明、中间产物格式、分支可达性、确认点 |
| Hybrid | 混合型，如 Pipeline+Generator | 兼具多种模式特征 | 识别主次模式，主模式决定审查重心，次模式决定补充检查项 |

无法归类的 skill：跳过模式专属检查，仅执行通用 D1/D2 扫描。

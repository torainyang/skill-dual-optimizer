# 示例：一次完整的 Skill 审查输出

> 以下是两个不同模式 skill 的审查示例，展示 Step 3 Plan 的完整输出格式。

---

## 示例 1：Generator 模式 — `report-generator`

# Skill 审查结论

## 模式判断
- 主模式：Generator
- 次模式：Pipeline
- 当前判断：模式匹配，但 Pipeline 特征（多步骤顺序执行）缺少阶段门槛

## 高优先级
- 〔A+B〕 description 只写了"生成报告"，缺少触发关键词（如"周报""月报""数据汇总"），导致触发率低
- 〔A〕 SKILL.md 第 45-80 行的报告模板（35 行）应拆到 `assets/report-template.md`，当前占用主文件 30% 的篇幅

## 中优先级
- 〔B〕 工作流缺少"数据源确认"步骤——用户未指定数据源时，skill 直接用默认数据源生成，可能产出错误报告
- 〔A〕 Gotchas 为空，建议至少补充：不要在用户未确认数据范围前生成最终报告

## 低优先级
- 〔B〕 输出报告缺少"生成时间"和"数据截止日期"字段，影响可追溯性

# 优化计划
1. 修改 `SKILL.md` frontmatter
   - 变更内容：description 改为"根据指定数据源生成结构化报告。当用户提到'生成报告''周报''月报''数据汇总''报表'时使用。"
   - 原因：提升触发准确性
   - 影响范围：仅本文件
   - 是否受用户指定方向驱动：否

2. 新建 `assets/report-template.md`
   - 变更内容：将 SKILL.md 第 45-80 行的报告模板移入
   - 原因：减少主文件篇幅，符合渐进式披露
   - 影响范围：需同步修改 SKILL.md（删除模板段落，增加链接）
   - 是否受用户指定方向驱动：否

3. 修改 `SKILL.md` 工作流
   - 变更内容：在 Step 1 后增加"Step 1.5: 确认数据源和时间范围"，设为硬性门槛
   - 原因：防止用默认数据源生成错误报告
   - 影响范围：仅本文件
   - 是否受用户指定方向驱动：否

4. 修改 `SKILL.md` Gotchas
   - 变更内容：增加"不要在用户未确认数据范围前生成最终报告"
   - 原因：补充高信号防护
   - 影响范围：仅本文件
   - 是否受用户指定方向驱动：否

请确认是否按以上计划执行。

## 示例 2：Tool Wrapper 模式 — `internal-api-helper`

# Skill 审查结论

## 模式判断
- 主模式：Tool Wrapper
- 次模式：无
- 当前判断：模式匹配，但知识组织方式有问题

## 高优先级
- 〔A〕 description 写的是"帮助使用内部 API"，过于笼统。应包含具体 API 名称或领域关键词（如"用户中心 API""权限校验接口""SOFARPC 调用示例"），否则与其他 Tool Wrapper skill 冲突
- 〔A〕 全部 API 文档（约 200 行）堆在 SKILL.md 正文，应拆到 `references/` 按模块分文件，SKILL.md 只保留索引和路由逻辑

## 中优先级
- 〔A+B〕 缺少 Gotchas。高频错误包括：用户问的是 v2 接口但 skill 只收录了 v1 文档；用户问的接口不在收录范围内时应明确告知而非猜测
- 〔A〕 没有版本标注——API 文档无日期或版本号，LLM 无法判断信息是否过时

## 低优先级
- 〔B〕 缺少"常见调用示例"，用户每次都要从零拼参数。补充 3-5 个高频场景的 curl/代码示例可显著提升体验

# 优化计划
1. 修改 `SKILL.md` frontmatter
   - 变更内容：description 改为"查询和使用内部用户中心 API。当用户提到'用户中心接口''权限校验 API''SOFARPC 用户服务'或具体接口名时使用。不适用于非用户中心的其他内部 API。"
   - 原因：精准触发 + 排除边界
   - 影响范围：仅本文件
   - 是否受用户指定方向驱动：否

2. 拆分 `references/`
   - 变更内容：将 SKILL.md 中的 API 文档按模块拆为 `references/user-api.md`、`references/auth-api.md`、`references/permission-api.md`
   - 原因：渐进式披露，按用户问题路由到对应文件
   - 影响范围：需同步修改 SKILL.md（删除文档段落，增加路由逻辑和链接）
   - 是否受用户指定方向驱动：否

3. 修改 `SKILL.md` Gotchas
   - 变更内容：增加"用户问的接口不在收录范围时，明确告知而非猜测参数""注意区分 v1/v2 接口版本"
   - 原因：防止 LLM 编造 API 参数
   - 影响范围：仅本文件
   - 是否受用户指定方向驱动：否

请确认是否按以上计划执行。

---

> **说明**：以上两个示例分别展示 Generator 和 Tool Wrapper 模式的审查输出。实际审查时根据目标 skill 的复杂度和模式调整篇幅。
